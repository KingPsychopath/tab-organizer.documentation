Tab Organizer Help/FAQ

What's the difference between a selected tab and a focused tab?

A selected tab is one that you have selected, for instance with Ctrl or Shift. These show up as green within Tab Organizer.

A focused tab is one that is currently selected within Chrome. These show up as tan/orange within Tab Organizer.

Does Tab Organizer support keyboard accessibility?

Absolutely! I put quite a bit of work into making sure that it is possible to do everything with the keyboard only.

Some things (like moving tabs) are more difficult with the keyboard, but it's still possible.

What are special queries?

Special queries are designed to make it easier to find certain types of tabs.

As an example, you can use the same:url special query to find all tabs that have the same URL.

How do you use special queries?

Just type them into the search box. You can also combine special queries and advanced syntax.

For instance, same:url OR google will find all tabs that have the same URL, and also the tabs that contain the word "google."

What are the different special queries?

  • is:favorite

    Shows the tabs that have been favorited. You can favorite/unfavorite a tab by clicking on the star on the left side of the tab.

  • is:image

    Shows the tabs that are images. Here is an example.

  • is:selected

    Shows tabs that have been selected (for instance, with Ctrl or Shift).

  • inurl:

    Shows tabs whose URL matches the supplied text. For instance, inurl:google.com would show tabs that have "google.com" in their URL.

  • intitle:

    Shows tabs whose title matches the supplied text. For instance, intitle:caturday would show tabs that have "caturday" in their title.

  • same:url

    Shows tabs whose URLs are the same. This is useful for finding duplicate tabs.

  • same:title

    Shows tabs whose titles are the same. This is useful for finding duplicate tabs.

  • window:

    Shows tabs which are within a certain window. For instance, window:5 would show all the tabs in the window named 5.

    You can also use a comma to easily show multiple windows at once: window:3,5,9

    And you can use a dash to show windows in sequence: window:1-5

    Note: this works with custom names too. If you rename a window "foo", you can show it with window:foo

    Note: there is a special value called focused that will show the window that is focused within Chrome.

    For instance, window:focused will show the focused window, and window:1-5,focused will show windows 1 through 5, and the focused window.

What's the point of inurl: and intitle:?

inurl: is useful when searching for things that are only found in URLs, like hash fragments or URL queries:

  • inurl:#
  • inurl:?
  • inurl:https://

intitle: is useful when you want to exclude things that normally appear in the URL. For instance, let's say you have a tab whose title contains the phrase "http".

If you typed in http it would probably match most of your tabs, because URLs often start with http:// On the other hand, you can use intitle:http to find the correct tab.

Another example is searching for the phrase "com". Websites often have that in the URL, so using intitle:com will filter those out.

You mentioned advanced syntax, what's that?

Advanced syntax is a way to combine words together to make your searches even more accurate, helping you to find the exact tabs you want.

If you have used Google Search, you may already know that it supports advanced syntax. Tab Organizer is very similar. Here is a list of the advanced syntax supported by Tab Organizer:

  • -

    Excludes an item from the search. For instance, -google would show all tabs that do not contain the word "google".

    This works with special queries as well. For instance, you can use -window:5 to show every window except the window named 5.

  • Space

    Using a space is the same as logical AND. This narrows your search to be more specific.

    For instance, wikipedia chrome would show tabs that match both "wikipedia" and "chrome" at the same time.

  • OR

    This adds items to the search. For instance, google OR youtube would show tabs that match either "google" or "youtube".

    You can also use the | character, like this: google | youtube

You can combine the above as much as you like. For instance, same:url google | window:bar -inurl:youtube would show the tabs that have the same URL and match "google", and also the tabs that are in the window named "bar", but not those that have "youtube" in their URL.

What has higher precedence: AND or OR?

AND has higher precedence. That means:

  • foo bar | qux

    (foo AND bar) OR qux

  • foo | bar qux

    foo OR (bar AND qux)

  • foo bar | qux corge

    (foo AND bar) OR (qux AND corge)
